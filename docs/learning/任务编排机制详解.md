## 1. 概述

OpenClaw 的任务编排机制是一个**多层次、可扩展的自动化系统**，它整合了以下核心能力：

### 1.1 核心功能

- **⏰ 定时任务调度**：基于 cron 表达式的周期性任务执行
- **🤖 Agent 自动化执行**：AI 驱动的自主任务处理
- **🔀 并发控制**：Lane 队列管理系统资源竞争
- **👥 子代理协作**：主代理派生子任务并行处理
- **🛡️ 容错机制**：指数退避重试、失败告警、超时保护

### 1.2 设计原则


### 3.3 调度流程

#### 3.3.1 定时器机制

```typescript
// src/cron/service/timer.ts

/**
 * 核心调度循环
 */
export async function onTimer(state: CronServiceState) {
  // 1. 防止并发执行
  if (state.running) {
    armRunningRecheckTimer(state);  // 重新武装定时器
    return;
  }
  
  state.running = true;
  
  try {
    // 2. 查找到期任务（加锁保证原子性）
    const dueJobs = await locked(state, async () => {
      await ensureLoaded(state, { forceReload: true });
      const now = state.deps.nowMs();
      const due = collectRunnableJobs(state, now);
      
      // 3. 标记为运行中（防止重复执行）
      for (const job of due) {
        job.state.runningAtMs = now;
      }
      await persist(state);
      
      return due;
    });
    
    // 4. 并发执行任务（受 maxConcurrentRuns 限制）
    const concurrency = resolveRunConcurrency(state);
    const results = await executeWithConcurrency(dueJobs, concurrency);
    
    // 5. 更新任务状态
    await locked(state, async () => {
      for (const result of results) {
        applyOutcomeToStoredJob(state, result);
      }
      await persist(state);
    });
    
  } finally {
    state.running = false;
    armTimer(state);  // 重新武装定时器
  }
}
```

#### 3.3.2 定时器武装逻辑

```typescript
export function armTimer(state: CronServiceState) {
  // 清除旧定时器
  if (state.timer) {
    clearTimeout(state.timer);
  }
  
  // 计算下次唤醒时间
  const nextAt = nextWakeAtMs(state);
  if (!nextAt) {
    return;  // 无待执行任务
  }
  
  const now = state.deps.nowMs();
  let delay = Math.max(nextAt - now, 0);
  
  // 安全下限：防止热循环（最小 2 秒）
  if (delay === 0) {
    delay = MIN_REFIRE_GAP_MS;  // 2000ms
  }
  
  // 上限：最多等待 60 秒（避免长时间不检查）
  const clampedDelay = Math.min(delay, MAX_TIMER_DELAY_MS);  // 60000ms
  
  // 设置定时器
  state.timer = setTimeout(() => {
    void onTimer(state);
  }, clampedDelay);
}
```

### 3.5 启动补偿机制

当 Gateway 重启时，可能错过了一些定时任务的执行窗口。Cron 系统提供了**启动补偿机制**：

```typescript
// src/cron/service/ops.ts

export async function start(state: CronServiceState) {
  // 1. 清理僵死的 runningAtMs 标记
  await locked(state, async () => {
    for (const job of jobs) {
      if (typeof job.state.runningAtMs === "number") {
        job.state.runningAtMs = undefined;  // 清除卡住的标记
        startupInterruptedJobIds.add(job.id);
      }
    }
  });
  
  // 2. 执行错过的任务（跳过刚才清理的任务）
  await runMissedJobs(state, { skipJobIds: startupInterruptedJobIds });
  
  // 3. 重新计算所有任务的 nextRunAtMs
  await locked(state, async () => {
    recomputeNextRuns(state);
    await persist(state);
    armTimer(state);
  });
}

/**
 * 执行错过的任务
 */
export async function runMissedJobs(state: CronServiceState, opts?) {
  const plan = await planStartupCatchup(state, opts);
  
  // 立即执行一部分（默认最多 5 个）
  const outcomes = await executeStartupCatchupPlan(state, plan);
  
  // 剩余的任务延迟执行（错峰执行，避免网关过载）
  await applyStartupCatchupOutcomes(state, plan, outcomes);
}
```

## 5. Lane 队列管理

### 5.1 什么是 Lane？

**Lane** 是 OpenClaw 的**并发控制机制**，将不同类型的任务分配到独立的队列中，避免资源竞争和优先级倒置。

```typescript
// src/process/lanes.ts

export enum CommandLane {
  MAIN = "main",           // 主会话（串行执行）
  SUBAGENT = "subagent",   // 子代理（最多 3 个并发）
  ACP = "acp",             // ACP 协议（串行）
  CRON = "cron",           // 定时任务（最多 2 个并发）
  CHAT = "chat",           // 聊天消息
  TOOL = "tool",           // 工具调用
}
```

### 5.2 Lane 并发配置

```typescript
// src/process/command-queue.ts

const LANE_CONCURRENCY = {
  main: 1,        // 主会话严格串行，保证顺序
  subagent: 3,    // 子代理允许适度并行
  acp: 1,         // ACP 串行
  cron: 2,        // 定时任务有限并行
  chat: 5,        // 聊天消息较高并发
  tool: 10,       // 工具调用高并发
};
```

### 5.3 队列实现

```typescript
// src/process/command-queue.ts

class CommandQueue {
  private queues = new Map<string, PQueue>();
  
  async enqueue<T>(
    lane: CommandLane,
    task: () => Promise<T>,
    opts?: EnqueueOptions
  ): Promise<T> {
    // 获取或创建队列
    let queue = this.queues.get(lane);
    if (!queue) {
      queue = new PQueue({
        concurrency: LANE_CONCURRENCY[lane] ?? 1,
      });
      this.queues.set(lane, queue);
    }
    
    // 加入队列执行
    return await queue.add(task, {
      priority: opts?.priority ?? 0,
      signal: opts?.signal,
    });
  }
}
```

## 7. 错误处理与重试

### 7.1 指数退避策略

Cron 任务执行失败时，采用**指数退避**策略进行重试：

```typescript
// src/cron/service/timer.ts

const DEFAULT_BACKOFF_SCHEDULE_MS = [
  30_000,    // 第 1 次错误 → 等待 30 秒
  60_000,    // 第 2 次错误 → 等待 1 分钟
  5 * 60_000,   // 第 3 次错误 → 等待 5 分钟
  15 * 60_000,  // 第 4 次错误 → 等待 15 分钟
  60 * 60_000,  // 第 5+ 次错误 → 等待 60 分钟
];

function errorBackoffMs(consecutiveErrors: number): number {
  const idx = Math.min(consecutiveErrors - 1, DEFAULT_BACKOFF_SCHEDULE_MS.length - 1);
  return DEFAULT_BACKOFF_SCHEDULE_MS[Math.max(0, idx)];
}
```

### 7.2 瞬态错误识别

系统能够识别**瞬态错误**（transient errors）并自动重试：

```typescript
const TRANSIENT_PATTERNS: Record<string, RegExp> = {
  rate_limit: /(rate[_ ]limit|too many requests|429)/i,
  overloaded: /\b529\b|\boverloaded\b|high demand/i,
  network: /(network|econnreset|econnrefused|fetch failed)/i,
  timeout: /(timeout|etimedout)/i,
  server_error: /\b5\d{2}\b/,  // 5xx 服务器错误
};

function isTransientCronError(error: string, retryOn?: CronRetryOn[]): boolean {
  return retryOn.some((key) => TRANSIENT_PATTERNS[key]?.test(error));
}
```

### 7.5 超时控制

每个任务都有**执行超时**限制，防止长时间阻塞：

```typescript
// src/cron/service/timeout-policy.ts

const DEFAULT_JOB_TIMEOUT_MS = 10 * 60_000;  // 默认 10 分钟

export function resolveCronJobTimeoutMs(job: CronJob): number {
  // 任务级别超时 > 全局配置 > 默认值
  return job.timeoutMs 
    ?? state.deps.cronConfig?.defaultTimeoutMs 
    ?? DEFAULT_JOB_TIMEOUT_MS;
}

// 执行带超时的任务
export async function executeJobCoreWithTimeout(state, job) {
  const timeoutMs = resolveCronJobTimeoutMs(job);
  
  const abortController = new AbortController();
  let timeoutId: NodeJS.Timeout;
  
  try {
    return await Promise.race([
      executeJobCore(state, job, abortController.signal),
      new Promise((_, reject) => {
        timeoutId = setTimeout(() => {
          abortController.abort("cron: job execution timed out");
          reject(new Error("cron: job execution timed out"));
        }, timeoutMs);
      }),
    ]);
  } finally {
    clearTimeout(timeoutId);
  }
}
```

## 8. 高级特性

### 8.1 维护模式

只读操作（如 `list`、`status`）使用**维护模式**，避免意外修改任务状态：

```typescript
// src/cron/service/jobs.ts

/**
 * 维护模式下的 nextRunAtMs 重计算
 * - 不会推进已过期的 nextRunAtMs（避免静默跳过任务）
 * - 仅修复损坏的时间戳
 */
export function recomputeNextRunsForMaintenance(
  state: CronServiceState,
  opts?: { recomputeExpired?: boolean }
): boolean {
  let changed = false;
  
  for (const job of state.store?.jobs ?? []) {
    if (!job.enabled) continue;
    
    const nextRun = job.state.nextRunAtMs;
    
    // 仅修复无效的时间戳
    if (typeof nextRun !== "number" || !Number.isFinite(nextRun)) {
      job.state.nextRunAtMs = computeJobNextRunAtMs(job, now);
      changed = true;
    }
    
    // 不推进已过期的 nextRunAtMs（除非显式要求）
    if (opts?.recomputeExpired && nextRun < now) {
      // 特殊处理...
    }
  }
  
  return changed;
}
```

### 8.2 会话清理

定期清理过期的 cron 运行会话，防止会话存储无限增长：

```typescript
// src/cron/session-reaper.ts

export async function sweepCronRunSessions(params: {
  sessionStorePath: string;
  nowMs: number;
  cronConfig?: CronConfig;
}): Promise<void> {
  const maxAgeMs = params.cronConfig?.sessionMaxAgeMs ?? 24 * 60 * 60_000;  // 默认 24 小时
  
  // 扫描会话存储
  const sessions = await loadSessions(params.sessionStorePath);
  
  for (const session of sessions) {
    // 清理过期的 cron 会话
    if (isCronSession(session) && session.lastActivityMs < nowMs - maxAgeMs) {
      await deleteSession(session.id);
    }
  }
}
```

### 8.3 防抖机制

防止任务在短时间内重复触发（例如 cron 表达式边界情况）：

```typescript
// src/cron/service/timer.ts

const MIN_REFIRE_GAP_MS = 2_000;  // 最小重触发间隔 2 秒

// 在计算下次运行时间时应用防抖
if (job.schedule.kind === "cron") {

```

### 8.4 心跳唤醒

Cron 任务可以触发**立即心跳唤醒**，确保 Agent 在线响应：


## 10. 最佳实践

### 10.1 任务设计原则

#### ✅ 推荐做法

1. **幂等性**：确保任务可以安全地重复执行
   ```typescript
   // ✅ 好的做法：检查是否已处理
   if (alreadyProcessed(today)) {
     return "今天已生成报告";
   }
   ```

2. **短小精悍**：单个任务专注于单一职责
   ```typescript
   // ✅ 拆分为多个小任务
   - "fetch-news"      → 获取新闻
   - "summarize-news"   → 摘要新闻
   - "deliver-news"     → 投递结果
   ```

3. **错误容忍**：优雅处理 transient errors
   ```typescript
   // ✅ 重试逻辑
   try {
     await fetchWithRetry(url, { maxRetries: 3 });
   } catch (err) {
     return `暂时无法获取数据: ${err.message}`;
   }
   ```

4. **超时保护**：设置合理的超时时间
   ```json
   {
     "timeoutMs": 300000  // 5 分钟，避免无限等待
   }
   ```

#### ❌ 避免的做法

1. **长时间运行的任务**：超过 10 分钟的任务应考虑异步化
2. **资源密集型操作**：避免在 cron 中执行大量计算
3. **硬编码凭证**：使用 secret 引用而非明文
4. **忽略错误处理**：始终处理可能的失败场景
