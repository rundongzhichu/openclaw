# OpenClaw 项目结构指引

本文档提供 OpenClaw 项目的完整结构说明,帮助开发者快速理解代码组织方式和核心文件功能。

## 📁 项目概览

OpenClaw 是一个运行在您自己设备上的个人 AI 助手,采用多通道架构,支持多种消息平台集成。项目使用 TypeScript 开发,基于 Node.js 运行时,采用 pnpm 作为包管理器。

### 技术栈

- **语言**: TypeScript (严格模式)
- **运行时**: Node.js >= 22.14.0 (推荐 Node 24)
- **包管理器**: pnpm@10.33.0
- **构建工具**: tsdown, Bun
- **测试框架**: Vitest
- **代码质量**: oxlint, oxfmt
- **通信协议**: WebSocket + RPC

## 🗂️ 目录结构详解

### 根目录核心文件

```
openclaw/
├── openclaw.mjs              # CLI 入口文件 - 命令行工具的启动脚本
├── package.json              # 项目配置 - 依赖、脚本、元数据
├── pnpm-workspace.yaml       # pnpm 工作区配置 - 管理 monorepo 结构
├── tsconfig.json             # TypeScript 编译配置
├── Dockerfile                # Docker 镜像构建配置
├── docker-compose.yml        # Docker Compose 编排配置
├── Makefile                  # Make 命令快捷方式
└── .npmrc                    # npm/pnpm 配置 - 扁平化依赖布局
```

#### 关键配置文件说明

**`openclaw.mjs`**
- CLI 二进制入口点
- 负责加载和启动 OpenClaw 命令系统
- 处理命令行参数解析和路由

**`package.json`**
- 定义项目元信息(名称、版本、描述)
- 声明所有运行时依赖和开发依赖
- 包含所有 npm scripts(构建、测试、运行等命令)
- 配置引擎要求(Node.js >= 22.14.0)
- 定义 pnpm 覆盖规则和补丁依赖

**`tsconfig.json`**
- TypeScript 编译器选项
- 启用严格类型检查
- 配置模块解析路径
- 定义输出目录和源映射设置

**`Dockerfile`**
- 多阶段构建配置
- 优化镜像大小和构建速度
- 包含生产环境运行时配置

#### 核心模块详细说明

**1. Gateway (网关)**
- **作用**: 整个系统的控制中心,处理所有客户端连接
- **关键文件**: `gateway/server.ts`, `gateway/auth-*.ts`
- **功能**: 
  - WebSocket 服务器监听(默认端口 18789)
  - 消息路由到对应通道和 Agent
  - 认证授权管理
  - 会话状态管理

**2. Channels (通道)**
- **作用**: 统一的消息渠道抽象层
- **关键文件**: `channels/channel-config.ts`, `channels/account-*.ts`
- **功能**:
  - 定义标准通道接口
  - 消息格式标准化
  - 账号和白名单管理
  - 支持 20+ 消息平台(WhatsApp, Telegram, Slack 等)

**3. Agents (代理运行时)**
- **作用**: AI Agent 的执行环境
- **关键文件**: `agents/agent-runtime-*.ts`, `agents/acp-spawn.ts`
- **功能**:
  - Pi Agent Runtime 集成
  - ACP 协议支持
  - Agent 作用域和超时管理
  - 工具流式输出

**4. Plugins (插件系统)**
- **作用**: 可扩展的插件架构
- **关键文件**: `plugins/bundle-*.ts`, `plugins/activation-*.ts`
- **功能**:
  - 插件打包和激活
  - 插件生命周期管理
  - 能力声明和发现
  - API 构建器

**5. Config (配置管理)**
- **作用**: 统一的配置管理系统
- **关键文件**: `config/agent-dirs.ts`, `config/channel-config*.ts`
- **功能**:
  - JSON/YAML 配置解析
  - Agent 和通道配置管理
  - 环境变量注入
  - 凭证安全管理

**6. Security (安全模块)**
- **作用**: 访问控制和隔离
- **关键文件**: `security/audit-*.ts`, `security/pairing.ts`
- **功能**:
  - DM 配对策略(未知发送者需要配对)
  - 审计和安全检查
  - API 密钥加密存储
  - 网络访问控制

**7. Cron (定时任务)**
- **作用**: 定时任务调度系统
- **关键文件**: `cron/scheduler.ts`
- **功能**:
  - Cron 表达式支持
  - 任务调度和执行
  - 并发控制
  - 错误重试机制

## 🔄 数据流和架构

### 消息处理流程

```
外部消息 (WhatsApp/Telegram/etc.)
    ↓
通道适配器 (extensions/*/)
    ↓
WebSocket 网关 (src/gateway/)
    ↓
消息路由器 (src/channels/)
    ↓
Agent 运行时 (src/agents/)
    ↓
LLM 提供商 (extensions/providers/*)
    ↓
工具调用 (src/agents/tools/)
    ↓
响应生成
    ↓
反向路由回原通道
    ↓
发送给用户
```

### 插件加载流程

```
启动时扫描 extensions/
    ↓
读取每个扩展的 package.json
    ↓
验证插件契约 (plugin-package-contract)
    ↓
加载插件入口 (index.ts)
    ↓
注册插件能力 (capabilities)
    ↓
初始化插件实例
    ↓
就绪可用
```

### 配置加载优先级

```
1. 内置默认值 (最低优先级)
2. 配置文件 (~/.openclaw/openclaw.json)
3. 环境变量 (OPENCLAW_*)
4. 命令行参数 (最高优先级)
```

## 📖 开发工作流

### 1. 环境准备

```bash
# 安装依赖
pnpm install

# 初始化配置
pnpm openclaw onboard --install-daemon

# 启动开发服务器
pnpm gateway:watch
```

### 2. 构建流程

```bash
# 编译 TypeScript
pnpm build

# 构建 Web UI
pnpm ui:build

# 构建 Docker 镜像
docker build .
```

### 3. 测试流程

```bash
# 运行所有测试
pnpm test

# 运行单元测试
pnpm test:unit

# 运行集成测试
pnpm test:integration

# 代码重复检测
pnpm jscpd
```

### 4. 代码质量

```bash
# 代码检查
pnpm lint

# 代码格式化
pnpm format

# 类型检查
pnpm tsgo:all
```

## 🔍 常见问题定位

### 网关无法启动

检查位置:
- `src/gateway/` - 网关实现和认证模块
- `~/.openclaw/openclaw.json` - 配置错误
- 日志: `~/.openclaw/logs/`

### 通道连接失败

检查位置:
- `extensions/<channel>/` - 通道实现
- `src/channels/` - 通道配置和管理
- 凭证配置: `~/.openclaw/credentials/`
- 网络代理设置

### 插件加载失败

检查位置:
- `src/plugins/` - 插件加载和激活逻辑
- 插件目录: `extensions/`
- 插件契约: `packages/plugin-package-contract/`

### 定时任务问题

检查位置:
- `src/cron/` - 定时任务调度器
- 任务配置文件
- 任务执行日志
