## 关键数据流

### 消息处理流程 (5 步)
```
1. 接收 → 2. 路由 → 3. 处理 → 4. 执行 → 5. 响应
   ↓         ↓         ↓         ↓         ↓
 Channel  Gateway   Agent     Tools    Channel
```

### Gateway 启动流程 (7 步)
```
CLI → 检查 → 初始化 → 加载插件 → 启动服务 → 注册渠道 → 就绪
```

### Agent 会话流程 (6 步)
```
创建 → 加载历史 → 构建 Prompt → LLM 调用 → 工具执行 → 返回结果
```

## 核心设计模式

### 1. 插件化架构
```typescript
// 所有功能都是插件
interface Plugin {
  id: string;
  setup: (ctx) => Promise<void>;
  poll?: (ctx) => Promise<void>;
}
```

### 2. WebSocket RPC
```
Client ←→ Gateway ←→ Agent Session
  │         │            │
  └─────────┴────────────┘
    双向实时通信
```

### 3. 会话隔离
```
Session A → Workspace A → Config A
Session B → Workspace B → Config B
Session C → Docker Sandbox → Restricted
```

## 关键特性

✅ **多通道支持**: 20+ 种消息渠道  
✅ **插件系统**: 热插拔扩展  
✅ **会话隔离**: Docker 沙箱  
✅ **故障转移**: 多模型备份  
✅ **安全认证**: Token + 配对码  
✅ **实时画布**: A2UI 渲染  
✅ **语音交互**: TTS/STT  
✅ **技能系统**: 可扩展工具  

## 性能指标

- **启动时间**: < 2s (Gateway)
- **消息延迟**: < 100ms (WebSocket)
- **并发会话**: 无限制 (独立 Session)
- **内存占用**: ~200MB (基础)
- **渠道数量**: 20+ (持续增加)

## 安全机制

🔒 **设备配对**: 首次连接需验证码  
🔒 **Token 认证**: JWT 签名验证  
🔒 **速率限制**: 防止滥用  
🔒 **权限控制**: Scope-based RBAC  
🔒 **审计日志**: 完整追踪  
🔒 **沙箱隔离**: Docker 限制

## 调试技巧

### 查看日志
```bash
# Gateway 详细日志
openclaw gateway --verbose

# 查看特定插件
OPENCLAW_LOG_PLUGINS=whatsapp openclaw gateway
```

### 诊断工具
```bash
# 健康检查
openclaw doctor

# 查看会话状态
openclaw sessions list
```
