# OpenClaw 代码中文注释指南

本文档说明了为 OpenClaw 项目代码添加中文注释的规范和已完成的文件。

## 📝 注释规范

### 1. 文件级注释 (File-level JSDoc)

每个核心代码文件顶部应包含详细的文件级注释，格式如下：

```typescript
/**
 * @fileoverview 文件功能概述
 * 
 * 详细描述文件的作用、功能和设计思路。
 * 可以包含：
 * - 核心功能列表
 * - 主要函数/类说明
 * - 使用示例
 * - 执行流程说明
 * - 数据结构定义
 * 
 * @module 模块路径
 */
```

### 2. 函数级注释 (Function-level JSDoc)

重要的导出函数应包含详细注释：

```typescript
/**
 * 函数功能描述
 * 
 * @param param1 - 参数说明
 * @param param2 - 参数说明
 * @returns 返回值说明
 * 
 * @example
 * // 使用示例
 * const result = await functionName(arg1, arg2);
 */
function functionName(param1: Type1, param2: Type2): ReturnType {
  // ...
}
```

### 3. 类注释 (Class-level JSDoc)

类和接口需要注释说明用途：

```typescript
/**
 * 类的功能描述
 * 
 * @example
 * const instance = new ClassName(config);
 * await instance.method();
 */
export class ClassName {
  // ...
}
```

### 4. 复杂逻辑注释 (Inline Comments)

对于复杂的业务逻辑，需要添加行内注释：

```typescript
// 步骤 1: 验证输入参数
if (!isValid(input)) {
  throw new Error("Invalid input");
}

// 步骤 2: 执行核心逻辑 (这里使用了策略模式)
const result = await strategy.execute(input);

// 步骤 3: 清理资源并返回结果
await cleanup();
return result;
```

## 📚 注释编写技巧

### 好的注释示例

✅ **清晰的功能描述**:
```typescript
/**
 * 子代理控制器 - 管理子代理的生命周期
 * 
 * 提供以下功能:
 * - 列出受控的子代理运行记录
 * - 调整子代理方向 (steer)
 * - 终止子代理运行
 * - 发送消息给子代理
 */
```

✅ **详细的参数说明**:
```typescript
/**
 * 孵化一个新的子代理
 * 
 * @param task - 子代理要执行的任务描述
 * @param label - 可选的标签，用于显示
 * @param agentId - 目标 Agent ID (默认为当前 Agent)
 * @param mode - 运行模式："run"(一次性) 或 "session"(持久化)
 * @param cleanup - 完成后清理策略："delete" 或 "keep"
 * @param sandbox - 沙箱模式："inherit" 或 "require"
 * 
 * @returns 孵化结果，包含子代理 session key 和状态
 * 
 * @throws Error 当超过最大子代理深度时抛出
 * 
 * @example
 * // 孵化一个一次性子代理
 * const result = await spawnSubagent({
 *   task: "分析这个代码库",
 *   mode: "run",
 *   cleanup: "keep"
 * });
 * console.log(`子代理已启动：${result.childSessionKey}`);
 */
```


### 避免的注释

❌ **重复代码的注释**:
```typescript
// ❌ 不好的注释
x = x + 1;  // 将 x 加 1

// ✅ 好的注释
// 递增重试计数器，用于指数退避算法
x = x + 1;
```

❌ **过时的注释**:
```typescript
// ❌ 不要写可能过时的注释
// 这个函数永远不会抛出异常 (但后来改了实现)
function safeOperation() {
  throw new Error("Oops");
}
```

❌ **无意义的注释**:
```typescript
// ❌ 废话注释
// 导入路径模块
import path from "path";

// ✅ 有用的注释
// 使用 node:path 确保跨平台路径处理兼容性
import path from "node:path";
```

## 🔧 工具和自动化

### 使用 ESLint 检查注释

```bash
# 检查缺少 JSDoc 的函数
pnpm lint --rule 'jsdoc/require-jsdoc: error'

# 检查无效的 JSDoc 参数
pnpm lint --rule 'jsdoc/check-param-names: error'
```

### 生成 API 文档

```bash
# 使用 TypeDoc 从注释生成文档
pnpm typedoc src/gateway/server.impl.ts \
  src/agents/agent-command.ts \
  src/agents/pi-embedded-runner/run.ts

# 输出到 docs/api 目录
```
